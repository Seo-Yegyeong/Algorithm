Data Structure(자료 구조)란?
=> 데이터를 저장하고, 검색하고, 수정하고, 삭제하는 데 사용되는 체계적인 형식.


[자료 구조의 종류]

● 선형 자료 구조
    1. 배열 (Array)
        같은 타입의 데이터를 연속된 공간에 저장
        인덱스를 통해 빠르게 접근 가능
        크기가 고정됨

    2. 연결 리스트 (Linked List)
        각 요소가 데이터와 다음 요소를 가리키는 포인터를 가짐
        삽입/삭제가 용이하지만, 접근은 느림

    3. 스택 (Stack)
        LIFO (Last In First Out)
        push, pop으로 작동
        예: 함수 호출 스택

    4. 큐 (Queue)
        FIFO (First In First Out)
        enqueue, dequeue
        예: 프린터 대기열

● 비선형 자료 구조
    5. 트리 (Tree)
        계층적인 구조 (부모-자식 관계)
        예: 이진 트리, 이진 탐색 트리 (BST), 힙, AVL 트리
    6. 그래프 (Graph)
        정점(Vertex)과 간선(Edge)으로 구성
        방향성 유무, 가중치 유무 등 다양한 종류 존재
        예: 지도, 소셜 네트워크

● 해시 구조
    7. 해시 테이블 (Hash Table / Hash Map)
        키-값 쌍으로 저장
        해시 함수를 통해 빠르게 접근
        충돌을 해결하는 기법 필요 (체이닝, 오픈 어드레싱 등)



[자료 구조 왜 배워야 하는가]
- 효율적인 시간/공간 복잡도를 위해
- 문제에 맞는 최적의 도구 선택
- 코딩 테스트나 기술 면접의 핵심
- 현실 세계의 문제를 더 잘 모델링하기 위해
